# 构建记录

这个目录用来记录 Knowledge Choice Feed 是怎么被 build 出来的。

它不只记录“做了什么功能”，更记录每一步背后的判断：

- 为什么从问答 Agent 转向内容服务平台；
- 为什么核心动作是“先选择，再揭示”；
- 为什么题卡质量 SOP 是第一功能；
- 为什么平台答案和用户评论前期必须分层；
- 为什么不能用 AI 伪造评论、点赞和高价值用户活跃；
- 哪些功能进入 MVP，哪些只是后续假设。

## 记录方式

每次重要迭代都按同一结构记录：

1. 当时的问题
2. 做出的判断
3. 关键修改
4. 产物链接
5. 尚未解决的问题
6. 下一步

## 维护规则

每次大改之后，必须更新 `how-i-built-knowledge-choice-feed.html`（补上这次的指令和它做了什么），并推送到 GitHub。构建记录不是一次性复盘，而是随产品一起生长的记忆。
